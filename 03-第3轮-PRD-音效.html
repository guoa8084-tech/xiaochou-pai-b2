第 3 轮 · PRD · 小丑牌配乐（音效系统）

5 个 SFX 事件 + BGM 循环 —— 接通第 1 轮已有的设置面板滑块

课程版本：B2 · 第 3 轮（音效子模块）/ 共 6 轮 · 文档版本：v1

轮次定位：轻量补丁，预算 25 分钟以内 第 1 轮已完成核心循环 + 动效 + 设置面板（含 BGM/SFX slider，值写入 localStorage 但未接音频）。本轮唯一任务：用 Howler.js 把音频文件加载起来，把 5 个事件时机挂上 SFX，把 BGM 跑起来，接通两个已有的音量 slider。不改任何游戏逻辑，不改 UI 布局。

一、一句话定位

为「小丑牌」网页游戏补全音效层——5 个 SFX 事件 + 1 轨 BGM，接通第 1 轮设置面板已有的两个音量 slider，让玩家在牌桌上获得完整的声画反馈。

二、目标用户与场景

维度描述
用户已经在玩第 1 轮产出版本的玩家（包括课程学员自测）
典型场景浏览器中玩牌，出牌时只有视觉动效、没有声音，感觉像「哑巴游戏」；通关时没有任何听觉庆祝反馈
当前解决方式无——第 1 轮 PRD §七明确将音频 defer 到第 3 轮
移动端场景玩家在手机上打开 GitHub Pages 链接，浏览器 autoplay 政策阻止音频自动播放；首次触摸屏幕时静默解锁，无需任何额外操作

三、核心价值主张

  • 声画同步：计分爆出、通关、失败这三个情绪高峰有对应的听觉强化，让「爽感」更完整
  • 零侵入接入：完全复用第 1 轮设置面板的 BGM / SFX slider（localStorage key balatro.settings 已存值），无需新增 UI，不改游戏逻辑
  • 静默降级：素材加载失败或 autoplay 解锁失败时，游戏继续正常运行，不报错、不阻塞

四、功能清单（MVP）

M1 · 5 个 SFX 事件 P0

以下 5 个事件必须在对应时机触发音效播放，均受 SFX 音量 slider 控制：

事件 ID触发时机（对应第 1 轮 §5.1 时间线步骤）文件命名建议推荐音色
sfx-select 玩家点击手牌切换选中 / 取消选中时（每次点击各触发一次） sfx-select.mp3 短促清脆点击音（<150ms），避免堆叠刺耳
sfx-play 玩家按下「出牌」按钮的瞬间（§5.1 步骤 1 飞牌开始前） sfx-play.mp3 卡牌打出声，有厚度，<300ms
sfx-score §5.1 步骤 5：chips × mult = score 大字在中央弹出的瞬间 sfx-score.mp3 爆裂 / 硬币叮当冲击音，0.5-1 秒，有冲击力
sfx-win 状态机切换到 won 时（3 关全通关） sfx-win.mp3 欢快升调胜利短曲，1-3 秒
sfx-lose 状态机切换到 lost 时（手数用尽未达目标分） sfx-lose.mp3 沉重降调失败音，1-2 秒
SFX 叠播上限：同时最多允许 4 个 SFX 实例并发播放。sfx-select 在快速连续点牌时最容易触发堆叠——若当前并发数已达上限，新触发的该音效直接丢弃（静默跳过），不排队等待。其他 4 个事件无需特殊处理（自然不会并发 4 次）。

M2 · BGM 循环播放 P0

项规则 锁定
文件命名bgm-main.mp3（单轨，覆盖整个游戏会话）
播放模式无限循环（loop: true），页面生命周期内不主动停止
关卡切换处理关卡通关 → 商店 → 下一关，BGM 不中断，继续循环；不做按关卡换曲（本轮只有一轨）
淡入淡出BGM 初次启动：淡入 800ms（避免突然爆音）；游戏结束（won/lost）时：BGM 淡出 1200ms 后停止，让 sfx-win / sfx-lose 的情绪不被遮盖
默认音量50（对应 localStorage bgmVolume: 50，0-100 映射到 Howler 0.0-1.0）

M3 · 设置面板接通 P0

第 1 轮已在 Modal 里做好两个 slider，并把值持久化到 localStorage。本轮的唯一工作是把这两个 slider 的 v-model 值实时 watch 后驱动 Howler 音量更新：

SliderlocalStorage 字段默认值接通行为
BGM 音量balatro.settings.bgmVolume50watch 变化 → bgmSound.volume(newVal / 100)
SFX 音量balatro.settings.sfxVolume70watch 变化 → 更新全局 SFX 播放音量基准（每次 play 时带入该值）
动画速度不影响音效播放速率（锁定决策）：动画速度 slider（慢/普通/快）改变的是 §5.1 动画时间线的全局倍率。音效在对应动画 step 触发，但音效本体的 rate 始终为 1.0，不跟随动画速度缩放。

M4 · 移动端 Autoplay 静默解锁 P1

浏览器 autoplay 政策要求必须有用户手势后才能播放音频。实现方式：

  • 监听全局 首次 pointerdown 或 click 事件（one-time listener，触发后立即 removeEventListener）
  • 在回调中调用 Howler.ctx.resume() 解锁 AudioContext，随后启动 BGM 淡入
  • 不显示任何 UI 提示（无 toast、无 banner）——玩家的第一次点击（通常是选牌）即完成解锁，完全无感

五、核心流程

页面加载 → Howler 预加载 6 个音频文件 → 等待首次 pointerdown → AudioContext 解锁 + BGM 淡入 → 游戏中各事件触发 SFX → 设置面板调 slider → 实时音量更新 → won/lost → BGM 淡出 + sfx-win/lose

音频初始化时序

  1. 组件挂载（onMounted）：调用 useAudio().init()，实例化 Howl 对象（5 个 SFX + 1 个 BGM）；此时不播放，仅预加载
  2. 读取 localStorage：从 balatro.settings 取 bgmVolume / sfxVolume，设置 Howl 初始音量
  3. 首次用户手势：one-time listener 触发 → resume AudioContext → BGM 开始 800ms 淡入循环
  4. 游戏事件发生时：对应时机调用 useAudio().play('sfx-select') 等方法
  5. 设置面板 slider 变化时：watch 触发 → 同步更新 Howl 实例音量（无需重新加载文件）

七、不做什么

以下内容本轮明确不做
  • 按关卡切换 BGM：单轨 BGM 贯穿全局，三个关卡不换曲——曲库扩充留给后续轮次
  • 解锁提示 UI（toast / banner）：静默解锁，不加任何视觉提示
  • Joker 触发专属音效：Joker 触发已有视觉高亮反馈（第 1 轮 §5.3），音效归入 sfx-score 的整体计分爆出时机，不单独为每张 Joker 拆分音效
  • 商店专属音效（买卡提示音、进商店音乐切换）：本轮商店静音，只靠视觉区分
  • 动画速度影响音效速率：音效播放速率始终 1×，与动画速度解耦（已锁定）
  • 音效可视化 / 频谱 / 均衡器：不在本课程范围
  • 粒子升级：通关撒金币、爆分粒子等高级视觉特效，见 04-第3轮-PRD-粒子.html

十、已知翻车点 / 性能上限 / 兜底策略 必读

以下每一条都是浏览器音频的已知坑，必须在实施阶段逐条覆盖

10.1 Autoplay 解锁失败

场景：AudioContext.resume() 因某些严格浏览器策略失败（返回 rejected Promise）。

兜底：catch 块里静默处理——console.warn('[Audio] autoplay unlock failed, running muted')，不抛错、不弹 alert、不阻止游戏继续。音量保持 0 但 Howl 实例照常存活，下次用户手势若再触发则重试一次 resume。

10.2 音频文件加载失败

场景：mp3 文件 404、CDN 超时、或素材路径配置错误。

兜底：Howler 的 onloaderror 回调触发时，console.warn('[Audio] failed to load: ' + src)，对应音效的 play() 调用改为 no-op（直接 return，不向上冒泡异常）。游戏逻辑、计分、状态机不受任何影响。

10.3 BGM 关卡切换硬切

场景：若未来要在关卡边界换 BGM，直接 stop / play 会产生刺耳硬切。

本轮处理：本轮只有 1 轨 BGM 不换，无此问题。为后续扩展留接口：封装 switchBgm(newSrc, fadeDuration = 800)，先 fade out 旧轨再 fade in 新轨。

10.4 SFX 叠播爆音

场景：玩家快速连续点选牌，sfx-select 在 200ms 内被触发 5+ 次，多个实例同时播放导致音量叠加失真。

兜底：维护一个运行时计数器 activeSfxCount，每次 play 前检查；若 >= 4 则 return（静默跳过新触发的 sfx-select）。Howl 的 onplay / onstop / onend 回调负责维护计数器。

10.5 Safari 移动端 AudioContext 状态

场景：iOS Safari 的 AudioContext 在页面 hidden（切后台）后可能自动 suspended，切回来后音频失效。

兜底：监听 document.visibilitychange；当 document.visibilityState === 'visible' 时调用 Howler.ctx?.resume() 重新激活。

10.6 文件命名与目录约定 锁定

balatro-game/src/assets/audio/
  ├── sfx-select.mp3     # 选牌音效
  ├── sfx-play.mp3       # 出牌音效
  ├── sfx-score.mp3      # 计分爆出音效
  ├── sfx-win.mp3        # 通关音效
  ├── sfx-lose.mp3       # 失败音效
  └── bgm-main.mp3       # BGM 主循环

全部采用 mp3 格式（兼容性最广，Howler 支持格式列表的首选）。文件大小建议：每个 SFX < 100KB，BGM < 3MB（128kbps 约 3 分钟曲目）。

10.7 构建产物包含音频文件

音频文件放在 src/assets/audio/ 目录下，Vite 默认会把 assets 目录下的静态资源打包到 dist/assets/。若文件体积超过 Vite 默认的 inline 阈值（4KB），会自动以 URL 引用形式处理——这对音频是期望行为（不要把 mp3 内联为 base64）。如有需要，在 vite.config.js 中把音频格式加入 assetsInclude 白名单：assetsInclude: ['**/*.mp3']。

十一、给全栈工程师的实施提示

本节是给工程师的技术建议，非产品需求，工程师可按实际情况微调实现方式

11.1 依赖安装

npm install howler
# 如果项目有 TypeScript，可选加：
npm install --save-dev @types/howler

11.2 推荐封装：Vue 3 Composable useAudio()

建议新建 src/composables/useAudio.js，集中管理所有 Howl 实例。核心职责：

  • init(bgmVolume, sfxVolume)：一次性实例化 6 个 Howl 对象，设置初始音量
  • play(eventId)：根据 eventId 查表触发对应 Howl.play()，内含 activeSfxCount 守卫
  • setBgmVolume(0-100) / setSfxVolume(0-100)：实时更新音量，供设置面板 watch 调用
  • unlockAudio()：封装 AudioContext.resume() + BGM 淡入逻辑
  • fadeOutBgm(duration)：game over 时淡出 BGM

11.3 设置面板接线方式

第 1 轮的设置面板组件里已有两个 slider 绑定到 settings.bgmVolume / settings.sfxVolume。只需在设置面板组件（或顶层 App.vue）中加两个 watch：

watch(() => settings.bgmVolume, (val) => audio.setBgmVolume(val))
watch(() => settings.sfxVolume, (val) => audio.setSfxVolume(val))
